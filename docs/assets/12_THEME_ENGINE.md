# THEME ENGINE

# Pit Wall Insight

Version: 1.0

Status: Living Document

---

# Purpose

The Theme Engine defines how Pit Wall Insight adapts to each Formula One constructor while maintaining a single, unified interface.

Themes should personalize the experience.

They should never redesign it.

Users should always recognize Pit Wall Insight regardless of the active constructor.

---

# Philosophy

Themes are personalities.

Not redesigns.

Every constructor shares

The same layout.

↓

The same typography.

↓

The same motion.

↓

The same spacing.

↓

The same interactions.

Only the visual identity evolves.

---

# Core Principle

Structure remains permanent.

Personality changes.

---

# Theme Hierarchy

Telemetry DNA

↓

Visual Language

↓

Layout

↓

Typography

↓

Motion

↓

Components

↓

Theme

Theme always comes last.

# Theme Elements

Each constructor controls

• Accent Color

• Secondary Accent

• Cursor

• Charts

• Hero Glow

• Ambient Lighting

• Interactive States

• Decorative Assets

Everything else remains identical.

No flashes.

No repaint artifacts.

# Persistence

Users should feel ownership of their workspace.

↓

# Recognition

Users should instantly recognize

Ferrari

Mercedes

McLaren

Williams

through atmosphere,

not through excessive branding.

---

# Do

✓ Preserve layout.

✓ Preserve hierarchy.

✓ Preserve motion.

✓ Preserve typography.

✓ Theme only what matters.

✓ Maintain consistency.

---

# Don't

✗ Redesign pages.

✗ Change spacing.

✗ Change fonts.

✗ Change animations.

✗ Introduce multiple accent colors.

✗ Create separate UI styles per constructor.

---

# Final Principle

The Theme Engine should feel like changing teams inside the same Formula One engineering platform.

The software remains the same.

Only the garage changes.
