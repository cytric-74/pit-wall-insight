# DATA STORYTELLING

# Pit Wall Insight

Version: 1.0

Status: Living Document

---

# Purpose

Pit Wall Insight is not a collection of charts.

It is a storytelling platform powered by data.

Every page should answer a question.

Every visualization should build understanding.

Every interaction should move the user closer to insight.

---

# Philosophy

Data without context is noise.

Visualization without purpose is decoration.

Analytics without explanation is confusion.

Every analytical experience should tell a story.

---

# Storytelling Hierarchy

Context

↓

Observation

↓

Comparison

↓

Explanation

↓

Insight

↓

Action

Never reverse this order.

---

# The First Question

Every screen should answer

"What am I looking at?"

before asking

"What does it mean?"

---

# The Second Question

After understanding

"What happened?"

the interface should naturally answer

"Why did it happen?"

---

# The Third Question

After explaining

"Why?"

the interface should answer

"What can I learn?"

---

# Information Pyramid

Context

↓

↓

Supporting Evidence

↓

Detailed Analysis

↓

Raw Data

Never expose raw information first.

"What data do we have?"

# Progressive Disclosure

Users should receive

Overview

↓

Summary

↓

Details

↓

Raw Data

Information unfolds naturally.

Never overwhelm immediately.

---

# KPI Rules

Every KPI should answer

One question.

Never combine multiple ideas.

Example

Fastest Lap

Good

Race Pace + Tyre Life + Fuel Usage

Bad

---

# Comparison Rules

Every comparison requires

Reference

↓

Difference

↓

Reason

↓

Conclusion

Comparisons without conclusions create confusion.

---

# Reading Flow

↓

Top Right

↓

Center

↓

↓

---

# Headlines

Every analytical section should contain a conclusion.

Example

"Ferrari gained race pace after Lap 32."

Not

"Race Pace Analysis"

Lead with insight.

Support with evidence.

---

# Annotation

Charts should explain important moments.

Examples

Safety Car

↓

Rain Begins

↓

Pit Stop

↓

DRS Enabled

↓

Fastest Lap

↓

Engine Failure

Never force users to remember events.

---

Every visualization should tell a story.

Pit Wall Insight does not visualize data.
