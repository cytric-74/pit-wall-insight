# CURSOR SYSTEM

# Pit Wall Insight

Version: 1.0

Status: Living Document

---

# Purpose

The cursor is one of the most frequently used interface elements.

Pit Wall Insight does not use a custom cursor.

It uses a Telemetry Probe.

The Telemetry Probe represents the user's connection with the engineering environment.

It should feel precise.

Not decorative.

---

# Philosophy

The cursor should never become the center of attention.

It should quietly reinforce the feeling of operating professional race engineering software.

The cursor communicates

• Precision

• Feedback

• Context

• Focus

• Interaction

It never exists for visual spectacle.

---

# Telemetry Probe

The default cursor is known as

Telemetry Probe.

It replaces the traditional cursor throughout the application except where native cursors improve usability.

---

# Design Language

The Telemetry Probe should resemble

• Laser Pointer

• Telemetry Sensor

• Engineering Probe

• Precision Instrument

Never

• Blob

• Glow Ball

• Neon Cursor

• Oversized Circle

• Particle Cursor

---

# Cursor Anatomy

The probe consists of

Core Dot

↓

Interaction Ring

↓

Context Layer

Only the required layers are visible.

---

# Default State

Small circular dot.

Perfectly round.

Minimal.

No glow.

No shadow.

No blur.

It should almost disappear.

---

# Idle Behavior

The probe remains perfectly stable.

No breathing.

No floating.

No pulsing.

The interface is calm.

---

# Follow Behavior

The cursor follows with slight interpolation.

Enough to feel physical.

Never enough to feel delayed.

Movement should resemble mechanical damping.

---

# Hover Philosophy

Hover should communicate

Recognition.

Not celebration.

The interface acknowledges the cursor.

It does not perform for it.

---

# Hover Sequence

Cursor approaches component

↓

Component detects proximity

↓

Cursor ring appears

↓

Border brightens

↓

Typography sharpens

↓

Interaction complete

---

# Magnetic Interaction

Interactive components attract the Telemetry Probe slightly.

Very subtle.

Never enough to alter usability.

The attraction should communicate

"this element is interactive."

---

# Button Hover

Probe expands slightly.

Thin ring appears.

Border brightens.

Typography increases contrast.

Shadow deepens slightly.

No scaling.

No glow explosion.

---

# Navigation Hover

Probe remains minimal.

Navigation text brightens.

Small underline appears.

Interaction should feel crisp.

---

# Card Hover

Probe remains centered.

Card border activates.

Accent line appears.

Chart preview subtly brightens.

Everything reacts together.

---

# KPI Hover

Probe activates.

Value increases emphasis.

Supporting metadata fades slightly.

The important number becomes the focus.

---

# Chart Hover

Default probe transforms into

Telemetry Crosshair.

The transition should be smooth.

No popping.

No flashing.

---

# Telemetry Crosshair

Consists of

Horizontal Guide

Vertical Guide

Center Probe

Current Coordinate

The crosshair should feel like engineering software.

---

# Crosshair Behavior

Moves smoothly.

Tracks data precisely.

Never snaps abruptly.

Always follows interpolation.

---

# Data Point Hover

Approaching a point should

Increase opacity

↓

Increase line thickness slightly

↓

Reveal tooltip

↓

Highlight corresponding KPI

The cursor becomes the trigger for the entire analytical system.

---

# Timeline Hover

Probe transforms into

Timeline Indicator.

Horizontal emphasis.

Supports race replay.

---

# Drag State

Probe becomes

Measurement Cursor.

Ring appears.

Movement indicator activates.

Dragging feels deliberate.

---

# Selection State

Ring contracts.

Center brightens.

Selection confirmed.

Duration remains short.

---

# Loading State

Cursor remains unchanged.

Never replace the cursor with loading spinners.

The interface communicates loading elsewhere.

---

# Disabled State

Cursor returns to native behavior.

No animation.

Reduced opacity.

---

# Text Selection

Always use native I-Beam.

Never replace it.

Readability is more important than branding.

---

# Input Fields

Use native text cursor.

Never interfere with text editing.

Precision always wins.

---

# Resize Areas

Use native resize cursors.

Do not override operating system expectations.

---

# Cursor Transition

Constructor changes should animate smoothly.

Never flash.

Never instantly swap colors.

---

# Motion

Cursor movement should always feel

Smooth

↓

Predictable

↓

Mechanical

↓

Fast

Never elastic.

Never delayed.

---

# Interaction Radius

The probe should begin responding slightly before entering an interactive element.

This creates anticipation.

Not surprise.

---

# Precision

Cursor position should always remain accurate.

Visual interpolation should never reduce click precision.

---

# Performance

Cursor rendering must remain at high frame rates.

It should never introduce latency.

---

# Accessibility

Users should be able to disable the Telemetry Probe.

Native cursor support should always exist.

Reduced motion settings should simplify cursor animations.

---

# Recognition

Users should recognize Pit Wall Insight by the cursor alone.

The Telemetry Probe should become part of the product identity.

---

# Do

✓ Keep the cursor small.

✓ Prioritize precision.

✓ Maintain consistency.

✓ Support context.

✓ Reinforce engineering.

✓ Keep animations subtle.

---

# Don't

✗ Giant cursors.

✗ Heavy glow.

✗ Neon effects.

✗ Random particles.

✗ Long cursor trails.

✗ Decorative animations.

✗ Laggy interpolation.

---

# Final Principle

The Telemetry Probe is not a visual effect.

It is an instrument.

Like every other part of Pit Wall Insight, it exists to help the user inspect, understand, and navigate information with confidence and precision.
