# ICONOGRAPHY

# Pit Wall Insight

Version: 1.0

Status: Living Document

---

# Purpose

Icons are supporting elements.

They guide.

They clarify.

They never become decoration.

Icons should disappear into the interface until needed.

---

# Philosophy

Icons should resemble symbols found inside

• Race Engineering Software

• Industrial Control Systems

• Aircraft Instrumentation

• Scientific Applications

They should never resemble

• Mobile App Emojis

• Gaming Interfaces

• Cartoon Graphics

• Decorative Illustrations

---

Every icon should feel

Technical

↓

Mechanical

↓

Minimal

↓

Precise

↓

Professional

---

# Design Principles

Icons should

Communicate immediately.

Scale perfectly.

Remain recognizable.

Support typography.

Never compete with data.

# Icon Style

Preferred

Filled Icons

Avoid

Two-tone Icons

Never

Never

3D Icons

Never

---

# Stroke

Thin.

Consistent.

Sharp.

Every icon should share identical stroke width.

---

# Corners

Softly rounded.

Never exaggerated.

Avoid playful geometry.

---

# Grid System

Every icon should be designed on a consistent grid.

Maintain identical visual weight.

Maintain optical balance.

---

# Visual Weight

Icons should never appear heavier than typography.

Typography remains the dominant visual element.

---

# Alignment

Icons align optically.

Not mathematically.

Always visually center icons.

---

# Do

✓ Maintain one icon style.

✓ Use outline icons.

✓ Keep strokes consistent.

✓ Animate subtly.

✓ Prioritize clarity.

---

# Don't

✗ Filled icon packs.

✗ Mixed icon libraries.

✗ Decorative icons.

✗ Emoji.

✗ Cartoon graphics.

✗ Inconsistent stroke widths.

---

# Recognition

Users should recognize

Pit Wall Insight

through

its minimal engineering-inspired iconography.

Icons should feel like controls on professional instrumentation.

# Final Principle

Icons are navigational instruments.

They guide attention.

They support understanding.

They should never become visual decoration.
